This page lists all the constant values and enumerations available in the PayNext SDK.

## Card Networks

Supported payment card networks for processing transactions.

<Accordion title="CardType enum values">
  The following card types are supported by the PayNext SDK:

  **Major Credit Cards:**
  - `visa` - Visa
  - `mastercard` - Mastercard  
  - `american-express` - American Express

  **Regional & Specialty Cards:**
  - `discover` - Discover
  - `jcb` - JCB (Japan Credit Bureau)
  - `diners-club` - Diners Club
  - `unionpay` - UnionPay (China)
  - `maestro` - Maestro (Debit)
</Accordion>

## Form Variants

Display modes for the checkout form component.

<Accordion title="Variant options">
  **Available variants:**

  - `default` - **Full form display** - All payment fields are visible immediately
  - Best for dedicated checkout pages
  - Provides clear visual hierarchy
  - Reduces cognitive load for users

  - `compact` - **Collapsible form** - Payment fields are hidden behind a "Card" button
  - Ideal for embedded checkout in smaller spaces
  - Saves screen real estate
  - Good for mobile-first designs
</Accordion>

## Error Types

Common error types that may be returned by the SDK.

<Accordion title="Error categories">

<Accordion title="Validation Errors">
  - `card_number_required` - Card number field is empty
  - `card_number_invalid` - Card number format is incorrect
  - `card_number_incomplete` - Card number is too short
  - `card_number_unsupported` - Card type not supported
</Accordion>

<Accordion title="Expiry Date Errors">
  - `expiry_required` - Expiry date field is empty
  - `expiry_invalid` - Expiry date format is incorrect
  - `expiry_incomplete` - Expiry date is incomplete
  - `expiry_expired` - Card has already expired
</Accordion>

<Accordion title="CVC Errors">
  - `cvc_required` - CVC field is empty
  - `cvc_incomplete` - CVC is too short
  - `cvc_invalid` - CVC format is incorrect
</Accordion>

<Accordion title="Cardholder Errors">
  - `name_required` - Cardholder name is required
  - `name_invalid` - Cardholder name format is invalid
</Accordion>

<Accordion title="Processing Errors">
  - `payment_failed` - General payment processing failure
  - `network_error` - Network connectivity issue
  - `authentication_failed` - Token authentication failed
</Accordion>

</Accordion>
